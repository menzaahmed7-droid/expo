This guide will show you how to add and customize a link preview for your app on iOS. A link preview is a feature commonly used on iOS to show user a preview of the content linked by the link.
If you have a link in your app, you can add preview by simply wrapping the link content with `<Link.Trigger>` component and adding `<Link.Preview>` component. This will create a preview of the page that the link points to.

```tsx
import { Link } from 'expo-router';

export default function Page() {
  return (
    <Link href="/about">
      <Link.Trigger>About</Link.Trigger>
      <Link.Preview />
    </Link>
  );
}
```

## New components

### Link.Trigger

A wrapper for the link's trigger. Anything that was previously passed as a child to `<Link>` should now be wrapped inside `<Link.Trigger>` when using `<Link.Preview>`.

### Link.Preview

Enables the link preview. You can customize its behavior and appearance. For more information see the section below on customizing previews.

```tsx
export default function Page() {
  return (
    <Link href="/about">
      <Link.Trigger>About</Link.Trigger>
      <Link.Preview />
    </Link>
  );
}
```

### Link.Menu & Link.MenuAction

Enables the context menu. Use `<Link.Menu>` as a container and `<Link.MenuAction>` for individual menu items.

```tsx
<Link href="/about">
  <Link.Trigger>About</Link.Trigger>
  <Link.Menu>
    <Link.MenuAction title="Share" icon="square.and.arrow.up" onPress={handleSharePress} />
    <Link.MenuAction title="Block" icon="nosign" destructive onPress={handleBlockPress} />
  </Link.Menu>
</Link>
```

## Customizing the preview

By default, the preview is rendered as a full-sized page snapshot. There are several ways to customize this behavior.

### Custom size

You can use the `width` and `height` props to suggest a preferred preview size. The system will consider these preferences but may override them based on available space or platform behavior.

```tsx
<Link href="...">
  <Link.Trigger>Content</Link.Trigger>
  <Link.Preview height="" />
</Link>
```

### Custom preview

If you don't want to show the default preview, you can pass custom content to the `<Link.Preview>` component via children. This custom content will replace the default preview of the link target.

```tsx
const [imageSize, setImageSize] = useState({ width: 0, height: 0 });
const { width } = useWindowDimensions();
const previewHeight = (width / imageSize.width) * imageSize.height;
<Link href="...">
  <Link.Trigger>Content</Link.Trigger>
  <Link.Preview width={screenWidth} height={previewHeight}>
    <Image
      onLoad={e => setImageSize(e.nativeEvent.source)}
      source={source}
      style={{ width: '100%', height: '100%' }}
    />
  </Link.Preview>
</Link>;
```

### Nested menus

You can nest menus by placing a `<Link.Menu>` inside another menu:

```jsx
<Link href="...">
  <Link.Trigger>About</Link.Trigger>
  <Link.Menu>
    <Link.MenuAction title="Share" icon="square.and.arrow.up" onPress={() => {}} />
    <Link.Menu title="More" icon="ellipsis">
      <Link.MenuAction title="Copy" icon="doc.on.doc" onPress={() => {}} />
      <Link.MenuAction title="Delete" icon="trash" destructive onPress={() => {}} />
    </Link.Menu>
  </Link.Menu>
</Link>
```

## Detecting if component is in preview

If you're building a component that might be rendered inside a preview, you can use the `useIsPreview()` hook to adjust its behavior accordingly:

```jsx
const isInsidePreview = useIsPreview();
// This will be true if component/screen is being rendered inside a preview
return isInsidePreview ? <Text>From Preview</Text> : <Text>I am outside of preview</Text>;
```

## Known limitations

### `replace` not Supported

The combination of link preview with `replace` mode is currently **not supported**. Previews can only be used with the default `push` navigation mode.

### JS Tabs and slots

When navigating within JS-driven tabs or slots, preview transition animations may appear clunky. This is due to React's delayed rendering while the native preview animation begins immediately. To prevent this issues use native tabs and stack.

### Missing `<Link.Trigger>`

If you render `<Link>` with a preview or context menu, but without `<Link.Trigger>`, an exception will be thrown. The same applies if you place any non-`Link.*` component directly inside `<Link>` when using preview mode.

### Multiple `<Link.Trigger>` children with `asChild` prop

When using `Link` with `asChild`, you may only specify **one** child for `<Link.Trigger>`. The `onPress` event will be forwarded to that child only.

### Changing href while preview is open

Changing the path in the `href` prop is **not supported**, while the preview is open. You may, however, modify the query parameters.
